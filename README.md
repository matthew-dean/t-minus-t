# TypeScript Minus TypeScript

**Status:** Early development

**What if you could write TypeScript, but never had to compile it, and it was shippable immediately?** What if it was still type-checked?

**TypeScript Minus TypeScript** is a VSCode/Cursor extension that lets you write TypeScript syntax in `.js` files. It stores your code as JavaScript with JSDoc comments, giving you TypeScript's developer experience and type checking while keeping your files as plain, immediately-runnable JavaScript.

```
What you write:          What gets saved:
let x: number = 5   →    /** @type {number} */
                         let x = 5
```

## Why?

- **No compilation step** - Your `.js` files are ready to run immediately
- **Full type checking** - TypeScript still validates your code via JSDoc
- **TypeScript syntax** - Write familiar TypeScript, not JSDoc by hand
- **Best of both worlds** - Type safety without the build complexity

## Features

- ✨ **TypeScript syntax in `.js` files** - Write `let x: number = 5` and it becomes JSDoc
- 🔄 **Live conversion** - See your changes reflected in real-time (with smart debouncing)
- 💬 **Smart comment handling** - Preserves your meaningful comments, converts type-only annotations
- 📝 **Auto-generate `.d.ts` files** - Optionally generate declaration files on save (off by default)
- 🎨 **Respects your formatting** - Maintains your indentation and code style

## Installation

### For Users (when published)

The extension will be available in the VSCode marketplace. Install it like any other extension.

### For Developers

1. Clone this repository
2. Run `pnpm install` from the root
3. Press `F5` in VSCode to launch the extension in a new window
4. Open a `.js` file
5. Run the command "TypeScript Minus TypeScript: Open as TypeScript"

## Usage

### Basic Usage

1. Open any `.js` file in VSCode/Cursor
2. Run the command `Typescript Minus TypeScript: Open as TypeScript` (or use the command palette)
3. Edit the file as if it were TypeScript
4. Save - your changes are automatically converted to JSDoc

### Settings

- `tMinusTS.autoGenerateDts` (default: `false`) - Automatically generate `.d.ts` files when saving `.js` files

### TypeScript Configuration

To enable type checking for your `.js` files, add a minimal `tsconfig.json` to your project:

```json
{
  "compilerOptions": {
    "checkJs": true
  }
}
```

This tells TypeScript to type-check your JavaScript files using the JSDoc comments. You can add other compiler options as needed, but `checkJs: true` is the minimum required for type checking.

For more details, see the [Extension README](packages/extension/README.md).

## How It Works

The extension uses VSCode's `TextDocumentContentProvider` API to create a virtual document view. When you "open as TypeScript", you're actually viewing a transformed version of your JavaScript file:

1. **Display**: JSDoc comments are converted to TypeScript syntax for editing
2. **Storage**: Your TypeScript edits are converted back to JSDoc when saved
3. **Comments**: Smart merging preserves your documentation while converting type annotations

## Supported TypeScript Features

The extension aims to support everything that can be expressed in JSDoc, as documented in the [TypeScript JSDoc Reference](https://www.typescriptlang.org/docs/handbook/jsdoc-supported-types.html):

- ✅ Basic types (`@type`)
- ✅ Function parameters (`@param`)
- ✅ Return types (`@returns`)
- ✅ Type definitions (`@typedef`)
- ✅ Generics (`@template`)
- ⚠️ Classes and interfaces (in progress)
- ⚠️ Advanced types (unions, intersections, etc.) (in progress)

## Development

### Prerequisites

- Node.js LTS (see `.nvmrc`)
- PNPM 10
- VSCode or Cursor

### Setup

```bash
# Install dependencies (installs for all packages)
pnpm install

# Build all packages
pnpm run build

# Run tests (converter package)
cd packages/converter && pnpm test

# Watch for changes (extension)
cd packages/extension && pnpm run watch

# Lint
pnpm run lint
```

### Project Structure

This is a monorepo using PNPM workspaces:

```
packages/
  converter/              # @t-minus-t/converter - Core conversion logic
    src/                  # TypeScript ↔ JSDoc converters
    test/                 # Comprehensive unit tests
  extension/              # VSCode extension
    src/                  # Extension code (uses converter package)
```

The converter package is independently testable and can be used in other projects.
